Decision rubric: Refactor vs Rebuild

Choose REBUILD only if ≥3 are true:

Framework mismatch to requirements (e.g., need SSR/islands but current app can’t support it sanely).

State management & routing are so entangled that auth/WS bugs can’t be isolated.

DB schema is fundamentally wrong (e.g., alerts not normalized; migrations would be riskier than a rewrite).

Integration tests <5% and logs indicate non-deterministic startup/race conditions that resist isolation.

Multiple circular-import/side-effect boot issues you can’t kill without ripping core files.

Otherwise: REFactor in place with Agent 3.
