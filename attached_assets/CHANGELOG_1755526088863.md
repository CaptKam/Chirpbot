## ChirpBot Development Laws

### Law #1: No New Engines Without Discussion
- Before adding any new sport engines or alert systems, we must discuss the approach first
- This prevents feature creep and ensures consistent architecture
- All new engines must follow established patterns and integrate with existing systems

### Law #2: Kid-Friendly Content Only
- All alert descriptions must be appropriate for all ages
- Use simple, clear language that anyone can understand
- Avoid complex jargon or technical terms
- Focus on excitement and fun rather than gambling or intense competition

### Law #3: User Settings Validation Required
- Every alert must check user settings before being sent
- If a user has disabled a specific alert type, that alert must not be sent
- This includes proper mapping between alert model types and user setting keys
- Failed validation should log the reason and silently block the alert

### Law #4: Deduplication Before Expensive Operations
- Always check deduplication BEFORE calling expensive services (AI analysis, external APIs)
- This saves costs and improves performance
- Log when alerts are blocked by deduplication to track system efficiency

### Law #5: AI Analysis Only When Enabled
- OpenAI analysis should only run when user has AI enabled in their settings
- When AI is disabled, use basic fallback descriptions
- This respects user preferences and saves API costs

### Law #6: Consistent Alert Structure
- All alerts must follow the same data structure across all sports
- Required fields: id, type, sport, title, description, gameInfo, timestamp, priority
- gameInfo must include: homeTeam, awayTeam, score, status, situation
- This ensures consistent display and processing across the frontend

### Law #7: Alert Format Consistency & 3-Second Readability
**VIOLATION OF THIS LAW REQUIRES REFUNDING DEVELOPMENT COSTS**
- **Title Format**: `[EMOJI] [ALERT TYPE] ([SCORE])` - NO team names, NO duplicate info
- **Description Format**: 3 lines max:
  - Line 1: Time/Period info
  - Line 2: Game situation
  - Line 3: Why it matters
- **No Duplicate Information**: Score, teams, time only mentioned ONCE across title + description
- **3-Second Rule**: User must understand the alert completely within 3 seconds
- **Team Names**: Only shown once in the gameInfo section, NEVER in title or description
- **Consistent Across All Sports**: Same format for MLB, NCAAF, NFL, NBA, NHL, CFL
- This law ensures optimal user experience and prevents information overload
