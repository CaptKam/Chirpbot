B) Live state (treat all “in-progress” variants as live)

Create/confirm a single helper used everywhere:

export function isLiveStatus(s: any) {
  const abs   = (s?.status?.abstractGameState ?? s?.abstractGameState ?? "").trim();
  const coded = (s?.status?.codedGameState ?? s?.codedGameState ?? "").trim();
  const text  = (s?.status?.detailedState ?? s?.detailedState ?? "").toLowerCase();
  return abs === "Live" || coded === "I" || text === "in progress" || text === "live";
}


Use this before polling play-by-play.

3) Make sure both sports emit via the same safety harness

Every alert must go through your single emitter (idempotent insert + WS broadcast). If you haven’t added it yet, keep this contract:

Compute dedup_key per alert.

Tennis: matchId:type:stablePointKey

MLB (plate-appearance scope): gamePk:type:inning:half:outs:bases:batterId:paId

Create dedup_hash = sha1(sport + ":" + dedup_key).

UPSERT on dedup_hash (ON CONFLICT DO NOTHING).

Broadcast over WS after DB insert succeeds.

This prevents dupes across BOTH sports and keeps delivery logic identical.
