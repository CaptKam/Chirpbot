Five Easy Laws (Replit-safe)
1) Only these top-level items exist

Allowed at root:

client/  server/  scripts/  config/  docs/  migrations/
.replit  replit.nix  package.json  pnpm-lock.yaml
README.md  .gitignore  tsconfig.json


Anything else = blocked.

2) One package manager, one lockfile

Use pnpm only. No npm/yarn lockfiles. Don’t let Replit “auto-install” different versions—pin toolchain via replit.nix.

3) Ignore Replit noise

Never commit Replit’s temp files or build output.

4) Guard commits

A tiny pre-commit hook rejects:

New top-level paths not on the allow-list

Multiple lockfiles

5) Protect main

No direct commits. Work on branches, then merge. (Even if you’re solo, this prevents half-configured changes hitting production.)

scripts/guard-new-files.js (super small)
// Blocks random top-level files/folders + multiple lockfiles
import { execSync } from 'node:child_process';

const ALLOWED_TOP = new Set([
  'client','server','scripts','config','docs','migrations',
  '.replit','replit.nix','package.json','pnpm-lock.yaml',
  'README.md','.gitignore','tsconfig.json'
]);

// new, untracked files
const added = execSync('git ls-files --others --exclude-standard', { encoding: 'utf8' })
  .split('\n').filter(Boolean);

// files staged for commit (rename/add)
const staged = execSync('git diff --cached --name-only', { encoding: 'utf8' })
  .split('\n').filter(Boolean);

const candidates = [...new Set([...added, ...staged])];

const offenders = candidates.filter(p => !ALLOWED_TOP.has(p.split('/')[0]));
if (offenders.length) {
  console.error('\n❌ Blocked files/folders not on allow-list:\n' + offenders.map(f => ' - ' + f).join('\n'));
  console.error('\nAllowed top-level items:\n' + [...ALLOWED_TOP].join(', ') + '\n');
  process.exit(1);
}

// lockfile sanity: only pnpm
const hasYarn = candidates.some(p => p.includes('yarn.lock'));
const hasNpm = candidates.some(p => p.includes('package-lock.json'));
if (hasYarn || hasNpm) {
  console.error('\n❌ Only pnpm is allowed. Remove yarn.lock / package-lock.json.\n');
  process.exit(1);
}

Optional (but handy)

Block direct commits to main
Create .husky/pre-push:

#!/bin/sh
branch="$(git rev-parse --abbrev-ref HEAD)"
if [ "$branch" = "main" ]; then
  echo "❌ Push to main is blocked. Create a feature branch and PR."
  exit 1
fi

30-second setup
pnpm i
pnpm prepare          # sets up Husky
git add .husky scripts .replit replit.nix .gitignore package.json
git commit -m "chore: add simple Replit guard rails"


That’s it. With these five laws + tiny guard:

Replit’s stray files won’t commit.

Toolchain/versions are pinned.

Only your approved structure lands in the repo.

Lockfile wars are impossible.

main stays clean.
