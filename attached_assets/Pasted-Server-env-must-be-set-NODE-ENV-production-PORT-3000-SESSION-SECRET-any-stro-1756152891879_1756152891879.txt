App feature flags (server-side)

Engine on: ALERTS_ENABLED=true

Sport on: SPORTS_ENABLED=MLB

Poll cadence: POLL_INTERVAL_MS=1500 (engine drops to 1000ms automatically in leverage)

AI enhancer: AI_ENABLED=true AI_RATE_LIMIT_MS=15000 AI_MIN_PRIORITY=80

Fixtures: leave unset or ALLOW_FIXTURES absent (prevents any dummy data)

(If these are DB-backed in your app, use the DB toggles below.)

Per-user (or global) DB toggles

Run against your Neon DB. Replace :uid with your user id (or run without where to make global defaults).

1) Enable channels

update settings set telegram_enabled = true where user_id = :uid;


2) Turn on all MLB alert types

update settings
set alert_types = coalesce(alert_types,'{}'::jsonb)
  || jsonb_build_object(
    'risp', true,
    'basesLoaded', true,
    'lateInnings', true,
    'closeGame', true,
    'powerHitter', true,
    'powerHitterOnDeck', true,
    'useRE24System', true,
    'RE24_L1', true,
    'RE24_L2', true,
    'RE24_L3', true
  )
where user_id = :uid;


3) AI enhancer settings

update ai_settings
set enabled = true,
    dry_run = false,
    model = 'gpt-4o-mini',
    rate_limit_ms = 15000,
    min_priority = 80
where sport = 'MLB';


4) Safety dedup windows (recommended)

-- Only if you store these in DB; adjust table/columns if needed
update dedup_rules set time_window_ms = 60000, realert_after_ms = 180000
where alert_type = 'RISP';
update dedup_rules set time_window_ms = 15000, escalation_only = true
where alert_type = 'POWER_HITTER_AT_BAT';
update dedup_rules set time_window_ms = 30000
where alert_type = 'POWER_HITTER_ON_DECK';
update dedup_rules set time_window_ms = 45000
where alert_type = 'BASES_LOADED';

Multi-source aggregator toggles

Primary MLB on: mlb_statsapi.enabled = true

Fallbacks: keep at least one (e.g., ESPN/TheSportsDB) enabled=true but lower priority.

Live detection: no toggle—just make sure code uses exact match (abstractGameState='Live' OR codedGameState='I').

Weather (only if you want wind boost tonight)

Set the two env vars above. No extra toggles needed. If you don’t set a key, the system will still alert—just without wind pill/boost.

Delivery & WS

WebSocket: ensure it’s mounted on the same HTTP server (no port 8080 split).

Ping: keepalive every 30s (already in code if you added it).

Telegram: telegram_enabled=true (per user) and both TELEGRAM_BOT_TOKEN + TELEGRAM_CHAT_ID present.

Indexes (one-time, for snappy UI)
create index if not exists idx_alerts_seen_time on alerts(seen, created_at desc);
create index if not exists idx_alerts_priority_time on alerts(priority desc, created_at desc);
