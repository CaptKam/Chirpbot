Highlighted Event (🔥 “Scoring chance: 75% run expectancy”)

Optional Context (e.g., player, weather, field position)

Bottom (Footer):

Confidence/Edge % (ex: “88% Confidence”)

Tagline / Edge type (💰 “Live Edge Alert”)

3. Content Style

Your problem may be that the text feels robotic or messy. Alerts should be short, sharp, and hype-driven:

❌ “Unknown QB/player ⚡ Indoor conditions • Unknown field position impact”

✅ “⚡ Inside the 20, QB under pressure — Red Zone drive”

Think sports ticker + betting edge tone.

4. Visual Design Tips

Spacing: Don’t cram text; group related info.

Icons/Emojis: Quick scanning beats words (🏟️, ⏰, 💰, 🔥).

Hierarchy: Bold the situation, keep context secondary.

Consistency: Every sport’s card should “feel” the same skeleton.

Render Rules (UI polish that just works)

Header row: icon + header.variant as title; right side: header.badge pill and metrics.confidence%.

Primary line: ai.text (truncate at 25 words; show “AI” pill when present).

Secondary line (context chips):

Clock chip (⏰), score chip (e.g., LAA 3 — NYY 4), state chip (inning/downs), weather chip if present (🌬️ 18 mph out).

Action row: left aligned “Open game”; overflow menu: “Mute this alert,” “Pin,” “Share.”

Colors: variant-based accent (e.g., RED_ZONE = red accent, BASES_LOADED = amber accent), neutral body.

Engine Integration (where to put the AI)

Cylinder fires → build the schema object without ai.text.

Call the AI Composer with the same object as JSON INPUT.

Validate output (≤ 25 words, no unknown player names).

Hydrate ai.text; compute lifecycle.dedupeKey; emit.

If AI fails: populate ai.text with deterministic fallback and keep moving.
