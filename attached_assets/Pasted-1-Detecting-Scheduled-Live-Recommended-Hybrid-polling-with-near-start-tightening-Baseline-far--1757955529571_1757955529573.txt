1) Detecting Scheduled → Live
Recommended: Hybrid polling with near-start tightening

Baseline (far from start): poll the calendar/status every 60s.

Pre-start window (T−10m → T+5m): tighten to every 10s.

Live-candidate confirmation: when status flips to “Live”, confirm once more after 3–5s (or require 2 consecutive reads) to avoid false starts from API jitter.

Extra signal (optional but nice): confirm with a secondary endpoint (e.g., linescore/clock started, inning/quarter > 0, or possession shown).

Why not pure WS from data vendors?

On Replit, long-lived sockets are flaky. Hybrid polling is simple and robust, but still feels real-time around game starts.

State machine (server)
SCHEDULED
  └─[status=Live confirmed twice within 10s]→ LIVE
LIVE
  ├─[status=Final confirmed twice]→ FINAL
  └─[status=Delayed/Suspended]→ PAUSED (keep light polling; engines idle)
PAUSED
  └─[status=Live reconfirmed]→ LIVE
FINAL → TERMINATE (clean up)

Practical guardrails

Timezone sanity: compute “now vs scheduledStart” using the venue’s TZ (or API-provided) not server local time.

Doubleheaders: distinguish by gameId (not team/date).

Delays/Suspensions: drop to 30–60s polling while paused; engines stay off.

2) Weather-sensitive cylinders (by sport)

Start with these; keep thresholds configurable.

MLB

Wind Shift Alert: Δwind ≥ 20° and speed ≥ 8 mph since last sample (directional change relevant to batter’s pull/oppo if you have that granularity).

High Outbound Wind: wind ≥ 12–15 mph blowing out to CF/LF/RF (venue-aligned bearing); boosts HR carry.

Inbound Wind Suppression: wind ≥ 12–15 mph blowing in; lowers long-ball risk (affects “HR live” confidence).

Precipitation Onset/Stop: rain starts/stops; impacts grip/command and delays.

Roof State Change (if retractable): open↔closed flip.

Extreme Temp Bands: <45°F or >90°F (ball flight / pitcher stamina context).

Humidity Spike: ΔRH ≥ 10% in 10–15 min window (minor, optional).

NFL / NCAAF / CFL

Sustained Wind: ≥ 15–20 mph; passing/kicking reliability down, rushing uptick.

Gusts: gust ≥ 25–30 mph; field goal & punt risk.

Precipitation Start/Stop: rain/snow begins/ends; fumbles/pace shifts.

Lightning/Weather Delay: stadium weather delay flag.

Extreme Cold/Heat: wind chill <20°F or heat index >95°F; pace/rotation notes.

WNBA / NBA (mostly indoor)

Generally ignore weather except roof leaks/venue advisories (rare). Safe to skip.

Cadence

Default while Live: every 60–120s.

When a weather-dependent cylinder is armed or recently triggered: tighten to 15–30s for 5–10 min, then relax.

3) Engine startup time when game goes Live
Targets (on Replit + polling)

Live detection: within ≤ 10s (baseline) or ≤ 3–5s during pre-start tightened window.

Engine spin-up: < 1s (it’s just scheduling timers + priming caches).

First alert latency (from Live): ≤ 2–3s for your first evaluation tick.

Pre-warm (lightweight, safe)

Keep engines OFF until Live, but “prime” them so they start instantly:

T−5 minutes: allocate engine context, load team/roster/venue meta, resolve API endpoints, create timers (paused).

T−2 minutes: lazy-fetch last-known linescore/clock once (no loops), and prep weather endpoint URL.

At Live confirmation: start the engine loop and start weather polling.

This gives you instant readiness without violating the “no heavy work until Live” rule.

5) Acceptance criteria (so you can test it)

Startup: no sports or weather API calls until calendar sync runs.

Scheduled games: sport & weather engines do not run.

Live transition: within ≤ 5s in pre-start window, otherwise ≤10s; engine + weather start immediately.

Weather-armed tightening: cadence drops to ≤ 30s within 2s of arming; returns to default after 10m idle.

Final: both engines stop within ≤ 5s; timers cleared; no further API calls.

Delays: status “Delayed/Suspended” puts engines in PAUSED; no polling of live feeds, only status every 30–60s.
