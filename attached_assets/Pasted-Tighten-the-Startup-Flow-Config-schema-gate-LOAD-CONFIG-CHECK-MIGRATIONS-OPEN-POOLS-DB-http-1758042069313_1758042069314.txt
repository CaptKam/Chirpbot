Tighten the Startup Flow

Config & schema gate: LOAD_CONFIG → CHECK_MIGRATIONS → OPEN_POOLS (DB/http) before anything else. Fail fast if either step isn’t green.

Warm caches: prime settings cache, team metadata, and AI prompt templates so first alerts aren’t cold.

Feature flags: lock demo seeding behind a flag/env to avoid accidental prod seeds.

Readiness vs liveness: export /healthz (liveness) and /readyz (readiness after calendar heartbeat + DB ping succeed).

Game Lifecycle: add guardrails

SCHEDULED → PREWARM

Pre-warm lightly: fetch lineups/umpire/venue + compile cylinder predicates; don’t start heavy polling.

Clock-skew handling: trust two signals (API status + first pitch/possession event) before LIVE.

LIVE

Debounce LIVE: require 2 consecutive confirmations in ≤5s window to avoid flapping.

Bulkheads: isolate per-sport workers so an MLB spike can’t starve NFL.

Backoff & circuit breakers for each upstream (stats, odds, weather).

Idempotency keys: <gameId>#<cylinder>#<frame> so retries don’t duplicate alerts.

Weather-on-Live: start with fast 2s cadence for first 5 minutes, then adaptive (e.g., 5–15s) unless conditions are volatile.

PAUSED / DELAYED (missing state)

Reduce to heartbeat polling; keep a minimal weather watch if the pause is weather-related.

OVERTIME / EXTRA INNINGS (edge)

Elevate cylinder priorities & shorten polling windows.

FINAL → COOLDOWN

Flush in-flight AI jobs, write a final summary alert (optional), archive state, release caches.

Keep a 90-sec tail to capture post-game feed corrections.

Alert Lifecycle: make it deterministic

Ingest (event) → Evaluate (cylinders) → Score → Filter (global/user) → Dedup → AI (optional) → Persist → Deliver

Explicit SLAs

Ingest→Evaluate: ≤150ms

Evaluate→Persist (no-AI): ≤300ms

AI path: fall back to non-AI if >400ms or model unavailable.

Scoring contract: standard 0–100 with sport-specific calibration; document thresholds.

Dedup BEFORE AI (you already noted this—good).

Priority queue: HIGH (scoring/late-game/state changes) vs NORMAL; deliver HIGH via SSE immediately, NORMAL in the next poll tick.

User gating: compile user settings to a bitmap/bitset at login so filtering is O(1).

Observability you’ll be glad you added

State transition logs with gameId, from, to, reason, latency_ms.

Per-cylinder metrics: fires/s, suppression count, average decision latency, top suppression reasons.

Missed-detection tracer: sampled replay buffer for 30s to diagnose “should have alerted” cases.

Weather diffs: log only 变化 (direction, gusts, precipitation onset) to cut noise.

Failure & recovery patterns

Dead letter queues for alert compose/persist failures.

At-least-once + idempotent writes to alerts table.

Cold-start budget: pre-compile engines on boot (one per sport) to avoid JIT hiccups at first pitch.
